## Phase 2: Full PGTR Forwarder

### 2.1 ŌĆö New Contract: `TaskMarketForwarder.sol`

Implements `IPGTRForwarder` (ERC-8194) ŌĆö this is TMP's concrete PGTR forwarder. The backend EOA calls this contract; it sets `pgtrSender`, transfers USDC on behalf of the payer, and forwards to `TaskMarket`.

```
packages/contracts/src/TaskMarketForwarder.sol
```

Key responsibilities:
1. Accept ERC-3009 `transferWithAuthorization` from payer ŌåÆ transfer USDC to `TaskMarket`
2. Verify receipt hash not consumed, not expired
3. Set `_pgtrSender = payer` (transient or storage, reset after call)
4. Call `TaskMarket.<function>(...)` ŌĆö functions now take no explicit requester param
5. Reset `_pgtrSender`
6. Emit `PaymentGatedCall(payer, taskMarket, selector, amount)`

Storage:
- `mapping(bytes32 => bool) public consumedReceipts`
- `address private _pgtrSender` (or `bytes32 transient` in Solidity Ōēź0.8.28)

USDC flow: `payer --ERC-3009--> TaskMarket escrow` (forwarder authorizes the transferWithAuthorization, TaskMarket escrow receives directly)
